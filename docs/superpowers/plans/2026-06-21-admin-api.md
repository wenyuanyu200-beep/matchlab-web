# Admin API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add JWT-protected administrator statistics, safe filtered lists, feedback access, and guarded user-role changes without affecting existing APIs.

**Architecture:** `middleware.RequireAdmin` enforces the JWT role after normal authentication. A focused `internal/admin` package separates API models, GORM persistence, business validation, and Gin handlers. The router mounts one `/api/admin` group with authentication and authorization middleware.

**Tech Stack:** Go 1.22, Gin, GORM, PostgreSQL, google/uuid, standard Go tests.

---

## File Structure

- Create `backend/internal/middleware/admin.go`, `admin_test.go`: role authorization middleware.
- Create `backend/internal/admin/model.go`: stats, list DTOs, filters and pagination.
- Create `backend/internal/admin/repository.go`: GORM counts, joins, safe selects and role updates.
- Create `backend/internal/admin/service.go`, `service_test.go`: validation and self-demotion protection.
- Create `backend/internal/admin/handler.go`, `handler_test.go`: HTTP parsing and responses.
- Modify `backend/internal/router/router.go`, `router_test.go`: admin dependency and six routes.
- Modify `docs/API_DOC.md`: admin curl and SQL setup flow.
- Do not modify `database/schema.sql`.
