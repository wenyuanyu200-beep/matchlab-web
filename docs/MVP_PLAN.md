# MatchLab 四天 MVP 计划

## 产品目标

在四天内完成可运行、可演示、可部署的最小闭环，而不是一次构建完整社交平台。

## Day 1：后端地基

- Gin 服务和环境配置
- `/api/health`
- PostgreSQL/GORM 连接入口
- 初始 schema
- systemd 与 Nginx 配置

验收：数据库不可用时 health 仍返回 HTTP 200；Ubuntu 可通过 systemd 启动。

## Day 2：最小认证

- 邮箱注册
- bcrypt 密码哈希
- 登录与 JWT access token
- 鉴权中间件
- 当前用户接口

验收：用户可以注册、登录，并使用 token 访问受保护接口。

## Day 3：活动与申请闭环

- 创建、浏览活动
- 申请活动
- 创建者接受或拒绝申请
- 最简匹配分数及说明

验收：两个演示账号可完成“发布—发现—申请—处理”的主流程。

## Day 4：轻量前端与部署

- Next.js 登录/注册页
- 活动列表、创建、详情和申请状态
- API 错误与加载状态
- 服务器部署、演示数据、冒烟测试

验收：公网浏览器可完整演示主流程。

## 暂不实现

- 微信、短信或第三方登录
- 即时聊天和推送
- 支付、会员与复杂运营后台
- 机器学习推荐
- 完整举报、审核、风控体系
- 原生 App 或微信小程序迁移
